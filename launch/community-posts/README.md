# Beam Community Launch Kit

This folder has ready-to-paste posts for fast distribution.
Use the active production host in all links (currently `https://beam-privacy.com`).
Use tracked campaign landing URLs (not the bare homepage) so acquisition attribution is preserved.

## Suggested Posting Order

1. `reddit-webdev.md` (highest developer reach)
2. `devto.md` (long-form tutorial, evergreen)
3. `indiehackers.md` (founder + traction feedback)
4. `reddit-saas.md` (pricing + positioning feedback)
5. `reddit-privacy.md` (privacy-focused audience)
6. `show-hn.md` (best posted Tue/Wed morning EST)

## Time Budget (about 25 minutes total)

- Reddit post: 3-4 minutes each (title + paste body + submit)
- Dev.to post: 6-8 minutes (format + publish)
- IndieHackers post: 4-5 minutes (post + add launch links)
- Show HN: 5-6 minutes (timing-sensitive)

## Posting Checklist

- Open the target platform URL from each file.
- Paste the suggested title.
- Paste the body as written (minor edits are fine).
- Keep each file's tracked campaign URL exactly as written (`ref` + `utm_*`).
- Submit and copy the live URL into `progress.txt`.
- Follow [launch-day-checklist.md](/home/scobb/repos/ralph-bootstrap/launch/launch-day-checklist.md) for posting order, exact URLs, and day-of monitoring.

## What To Monitor Right After Posting

- Comments in the first 30 minutes (respond quickly).
- Signup starts and completions in Beam funnel events.
- Acquisition source/campaign lift in `/dashboard/acquisition` (default external-only view).
- Traffic source and channel lift in `/dashboard/sites/:id/analytics`.
- Any confusion in comments that should become FAQ or docs updates.
